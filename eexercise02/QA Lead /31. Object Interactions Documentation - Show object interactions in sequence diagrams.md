## Object Interactions Documentation  
### Object Interactions in Sequence Diagrams

---

## 1️.Object Interaction: User Registration & Login

### Participating Objects
- User
- WebInterface
- AuthenticationService
- Database

### Interaction Description
1. The **User object** sends registration details to the **WebInterface object**.
2. The **WebInterface** forwards the request to the **AuthenticationService** for validation.
3. The **AuthenticationService** stores user data in the **Database**.
4. After successful registration, a confirmation is returned to the **User**.
5. During login, the **User** submits credentials.
6. The **AuthenticationService** verifies credentials using the **Database**.
7. On success, the **User** is granted access to the dashboard.

### Interaction Type
- Request–Response
- Synchronous communication

---

## 2️.Object Interaction: Passport Application Submission

### Participating Objects
- User
- ApplicationPortal
- PassportApplicationService
- Database

### Interaction Description
1. The **User object** initiates application submission.
2. The **ApplicationPortal** sends application data to the **PassportApplicationService**.
3. The service stores application details in the **Database**.
4. The generated application ID is returned to the **User**.

### Interaction Type
- Data transfer
- Transaction-based interaction

---

## 3️.Object Interaction: Document Upload & Verification

### Participating Objects
- User
- DocumentModule
- DocumentService
- PassportOfficer
- Database

### Interaction Description
1. The **User** uploads documents through the **DocumentModule**.
2. The **DocumentService** stores documents in the **Database**.
3. The **PassportOfficer** accesses documents for verification.
4. Verification status is updated in the **Database**.
5. The system notifies completion of verification.

### Interaction Type
- Multi-object interaction
- Role-based access interaction

---

## 4️.Object Interaction: Police Verification & Passport Approval

### Participating Objects
- PassportOfficer
- PoliceDepartment
- ApplicationService
- Database
- User

### Interaction Description
1. The **PassportOfficer** requests verification from the **PoliceDepartment**.
2. The **PoliceDepartment** submits a verification report.
3. The **ApplicationService** updates application status.
4. The **Database** stores the final approval status.
5. The **User** receives passport approval notification.

### Interaction Type
- Inter-departmental communication
- Event-driven interaction
