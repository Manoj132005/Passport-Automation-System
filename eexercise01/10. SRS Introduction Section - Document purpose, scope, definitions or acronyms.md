# SRS Introduction

## 1. Purpose
The purpose of this Software Requirements Specification (SRS) document is to clearly define the functional and non-functional requirements of the Passport Automation System.  
It serves as a reference for developers, testers, project managers, and stakeholders to understand what the system should do.

## 2. Scope
The Passport Automation System is a web-based application that automates the process of passport application, verification, appointment scheduling, and approval.  
It allows citizens to apply online, upload documents, and track application status, while enabling passport officers and administrators to verify and process applications efficiently.

## 3. Definitions, Acronyms, and Abbreviations
- PAS – Passport Automation System  
- Applicant – A citizen applying for a passport  
- Admin – System administrator who manages the system  
- Verification – Process of checking submitted documents  
- SRS – Software Requirements Specification
